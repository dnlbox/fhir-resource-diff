# Spec 35 ‚Äî Add MedicationUsage to the R5 resource registry

## Goal

In FHIR R5, `MedicationStatement` was renamed to `MedicationUsage`. The resource registry
(`src/core/resource-registry.ts`) does not include `MedicationUsage`, causing a false warning when
validating R5 resources generated by fhir-test-data or any other R5-compliant tool:

```
‚öÝ resourceType: resourceType 'MedicationUsage' is not in the known registry for R5.
  This may be valid ‚Äî check https://hl7.org/fhir/resourcelist.html
```

This is a false positive ‚Äî `MedicationUsage` is a valid and normative R5 resource.

## Dependencies

- Spec 13 (resource-type-registry) ‚Äî complete
- Spec 17 (version-aware-validation) ‚Äî complete

## Deliverables

- `src/core/resource-registry.ts` ‚Äî add `MedicationUsage` entry restricted to R5 only
- `tests/core/resource-registry.test.ts` ‚Äî verify MedicationUsage is valid in R5 and invalid in R4/R4B
- `CHANGELOG.md` ‚Äî add entry under `[Unreleased]`

## Key interfaces / signatures

New registry entry shape (matches existing ResourceTypeInfo structure):

```typescript
{
  resourceType: 'MedicationUsage',
  category: 'clinical',
  versions: ['R5'],
  description: 'Record of a patient using or having used a medication (renamed from MedicationStatement in R5).',
  maturityLevel: 2,
  useCases: [
    'Recording current medications during admission',
    'Documenting medication reconciliation',
    'Tracking long-term medication use',
  ],
  keyFields: [
    { field: 'status', note: 'Required. recorded | entered-in-error | draft' },
    { field: 'medication', note: 'Required in R5. Replaced medicationCodeableConcept/Reference with polymorphic medication.concept and medication.reference' },
    { field: 'subject', note: 'Required. Reference to the patient' },
  ],
  versionNotes: {
    'R4‚ÜíR5': 'Renamed from MedicationStatement. medication field changed to polymorphic structure.',
  },
},
```

Also add a `versionNotes` entry to `MedicationStatement` indicating it was superseded in R5:

```typescript
versionNotes: {
  'R4B‚ÜíR5': 'Renamed to MedicationUsage in R5. MedicationStatement does not exist in R5.',
},
```

## Implementation notes

- `MedicationUsage` exists **only in R5** ‚Äî do not add it to R4 or R4B versions
- `MedicationStatement` exists in R4 and R4B but **not in R5** ‚Äî the existing registry entry
  should be updated to reflect `versions: ['R4', 'R4B']` only
- The fhir-required-fields rule should also handle MedicationUsage ‚Äî add the R5 required fields
  (`status`, `medication`, `subject`) to the required fields map
- The fhir-status-values rule should include MedicationUsage status values for R5:
  `recorded | entered-in-error | draft`

## Acceptance criteria

```bash
# MedicationUsage must be valid in R5 with no warnings
echo '{"resourceType":"MedicationUsage","id":"test-01","status":"recorded","medication":{"concept":{"coding":[{"system":"http://snomed.info/sct","code":"372687004","display":"Amoxicillin"}]}},"subject":{"reference":"Patient/test-patient"}}' \
  | fhir-resource-diff validate - --fhir-version R5
# Expected: valid

# MedicationUsage must warn/error in R4 (unknown resource type for R4)
echo '{"resourceType":"MedicationUsage","id":"test-01"}' \
  | fhir-resource-diff validate - --fhir-version R4
# Expected: warning about unknown resourceType for R4

# fhir-resource-diff info MedicationUsage must return R5 info
fhir-resource-diff info MedicationUsage
# Expected: info shown, not "Unknown resource type"

# End-to-end with fhir-test-data
fhir-test-data generate medication-statement --locale us --seed 1 --no-pretty --fhir-version R5 \
  | fhir-resource-diff validate - --fhir-version R5
# Expected: valid (no MedicationUsage registry warning)
```

## Do not do

- Do not add MedicationUsage to R4 or R4B ‚Äî it doesn't exist there
- Do not remove MedicationStatement from the registry ‚Äî it's valid in R4 and R4B
- Do not add full StructureDefinition validation ‚Äî curated fields only
