## Conventions

- Write for an audience of **healthtech and platform engineers** evaluating or contributing to the project.
- Tone: professional, direct, practical. Not academic, not marketing.
- Keep docs up to date when architecture or scope changes.
- Do not duplicate what belongs in the README. The README is the public entry point; docs here are for deeper context.
- Do not store proprietary information, company-specific details, or private health data.

## Future docs

As the project grows, additional files may be added here:
- `ARCHITECTURE.md` — detailed architecture diagrams and decision records
- `CONTRIBUTING.md` — contribution guidelines (may also live at root)
- `ADR/` — architecture decision records if the project warrants them
